# Deep Learn Scout — 广度+深度学习侦察兵

## 输出合约

每次运行必须产出：
1. **领域综述**（300-500字，包含定义、现状、核心方向）
2. **资源库**（4个分类，每类3-8条，含评分）
3. **学习路线图**（阶段划分 + 每阶段重点资源）
4. **搜索日志**（记录广度/深度搜索的 query 和来源）

---

## 工作流（5个阶段，顺序执行）

### Phase 1 — 理解目标 & 拆解关键词

收集这些信息（若用户未明确提供，**主动询问背景水平**后再继续）：

- **领域主题**：用户输入的主题（必填）
- **背景水平**：入门 / 有基础 / 进阶（**必须确认，影响资源筛选逻辑**）
- **关注重点**：理论 / 工程实践 / 研究前沿（默认：全部）
- **语言偏好**：中文 / 英文 / 双语（默认：双语）

**关注重点影响搜索策略：**
- `工程实践`：Phase 2B 课程搜索权重加倍，Phase 3 论文精简到3篇
- `研究前沿`：Phase 2B 课程可跳过（进阶用户），Phase 3 论文扩展到10篇
- `理论`：均衡执行，教材/综述优先

生成 3-5 个搜索关键词组合，覆盖：核心概念、应用方向、主流框架/工具、最新进展。

---

### Phase 2 — 广度搜索（Breadth Scan）

**目标**：快速覆盖领域的宽度，收集各类资源入口。

并行执行以下 4 类搜索（使用 Exa MCP 或 WebSearch）：

#### 2A. 开源项目搜索
```
query: "{topic} github awesome OR top projects site:github.com"
query: "{topic} 开源项目 star 排名"
```
收集：repo 名称、star 数、简介、适用场景

#### 2B. 课程/教程搜索
```
query: "{topic} course tutorial 2024 2025 beginner intermediate"
query: "{topic} 课程 视频教程 学习路线 bilibili OR coursera OR udemy"
```
收集：平台、时长、难度、评价

#### 2C. 最新资讯/博客
```
query: "{topic} latest news trends 2025"
query: "{topic} 技术博客 综述 知乎 OR medium site:medium.com"
```
收集：发布时间、核心观点摘要

#### 2D. 社区/讨论热点
```
query: "{topic} reddit discussion community best resources"
query: "{topic} 学习路线 经验 推荐"
```
收集：社区共识、避坑建议

**输出**：原始资源列表（20-40条），带来源URL。

---

### Phase 3 — 深度挖掘（Depth Dive）

**目标**：从广度结果中识别核心论文，萃取学术核心观点。

#### 3A. 找核心论文

按以下优先级识别5-10篇关键论文：

**优先级 1（必选）**：Survey / Review 综述类（覆盖全局视野）
**优先级 2**：引用量 > 100 的奠基性方法论文
**优先级 3**：2024/2025 发表的最新 SOTA 论文
**优先级 4**：广度搜索中被多次提及的论文

补充论文搜索：
```
query: "{topic} survey paper arxiv 2023 2024 2025"
query: "{topic} foundational paper landmark"
query: "site:arxiv.org {topic} review comprehensive"
```

**去重规则**：同一作者同一工作的不同版本只保留最终版。
**上限**：每个优先级最多选3篇，总数不超过10篇。

#### 3B. 论文核心观点萃取

对每篇论文提取（若无全文则基于摘要+标题）：
- 解决的核心问题
- 主要贡献/创新点（1-3条）
- 关键结论/实验结果
- 局限性或争议点
- 引用量/影响力指标（若可获取）

#### 3C. 知识点聚类

将所有资源涉及的知识点归纳为 3-5 个核心概念簇，
每簇写一句话说明与整体领域的关系。

---

### Phase 4 — 评分 & 排序

对所有收集的资源按以下维度打分（1-5分）：

| 维度 | 说明 | 权重 |
|---|---|---|
| 质量/权威性 | 来源可信度、作者背景、引用量 | 30% |
| 实用性 | 可直接上手的程度 | 25% |
| 新鲜度 | 发布/更新时间 | 20% |
| 适配度 | 与用户背景水平匹配 | 15% |
| 社区认可 | star数/评价/讨论热度 | 10% |

计算加权综合分，每类资源取Top 5。

---

### Phase 5 — 整合输出文档

按 `references/output-template.md` 格式生成最终文档。

核心结构：
1. 领域综述
2. 资源库（项目 / 课程 / 论文 / 资讯）
3. 评分排行榜
4. 学习路线图（ASCII 或 Mermaid）
5. 快速启动建议（Next 3 actions）

---

## 失败处理

| 场景 | 处理方式 |
|---|---|
| Exa/WebSearch 不可用 | 降级到 WebFetch 手动搜索主要站点 |
| 子 agent 中 WebSearch 受限 | 所有搜索在主会话中执行，不使用子 agent 执行搜索类任务 |
| 论文无法访问全文 | 仅基于摘要萃取，标注「摘要摘录」 |
| 资源数量不足（<10条） | 扩大搜索词范围，降低筛选门槛并标注 |
| 主题过宽（如"AI"） | 请求用户缩窄到子领域后继续 |
| 主题过窄（无结果） | 自动扩展到上位概念，提示用户 |

## 验证检查清单

运行完成前确认：
- [ ] 广度资源 ≥ 4 类，每类 ≥ 3 条
- [ ] 深度论文 ≥ 5 篇，每篇有核心观点（含「解决问题」+「核心贡献」）
- [ ] 所有资源有综合评分（必须用 scoring-rubric.md 计算）
- [ ] 学习路线图有阶段划分（≥ 3 阶段，每阶段绑定具体资源链接）
- [ ] 文档有综述介绍（≥ 200字，含定义/现状/核心方向）
- [ ] 每条资源有 URL（或说明无法获取原因）
- [ ] 文档末尾有搜索日志（每个 phase 的 query 记录）
- [ ] 全局 Top 10 排行榜已生成
