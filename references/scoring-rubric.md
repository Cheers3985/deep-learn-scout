# 资源评分标准手册

## 评分维度详解

### 1. 质量/权威性（30%）

| 分数 | 判断标准 |
|---|---|
| 5 | 顶会论文(NeurIPS/ICML/CVPR等) / 官方文档 / 领域权威作者 / 引用量>500 |
| 4 | 知名机构出品 / GitHub官方教程 / 引用量100-500 |
| 3 | 有同行评审 / 知名博主 / GitHub star>1k |
| 2 | 个人博客但内容扎实 / star 100-1k |
| 1 | 来源不明 / 内容浅薄 / 无法验证准确性 |

### 2. 实用性（25%）

| 分数 | 判断标准 |
|---|---|
| 5 | 有完整代码+数据集+说明文档，可直接复现 |
| 4 | 有代码，需少量调整 |
| 3 | 有伪代码或部分代码，需自行实现 |
| 2 | 纯理论，无代码 |
| 1 | 理论不完整，无法指导实践 |

### 3. 新鲜度（20%）

| 分数 | 时间范围 | 说明 |
|---|---|---|
| 5 | 2024-2025 | 最新进展 |
| 4 | 2022-2023 | 较新 |
| 3 | 2020-2021 | 中期 |
| 2 | 2018-2019 | 稍旧 |
| 1 | 2017及以前 | 历史参考（经典论文除外，经典给4分） |

> **经典论文例外规则**：若为领域奠基性工作（如 Transformer、ResNet），新鲜度固定给4分。

### 4. 适配度（15%）

根据用户背景水平评分：

| 分数 | 入门用户 | 有基础用户 | 进阶用户 |
|---|---|---|---|
| 5 | 步骤详细，前置要求少 | 深度适中，有进阶内容 | 前沿/工程细节充足 |
| 3 | 内容适中但有跳跃 | 略浅或略深 | 偏基础 |
| 1 | 严重超出或远低于水平 | 同左 | 同左 |

### 5. 社区认可（10%）

| 分数 | GitHub Stars | 论文引用 | 课程评分 |
|---|---|---|---|
| 5 | >10k | >1000 | >4.5/5 |
| 4 | 1k-10k | 100-1000 | 4.0-4.5 |
| 3 | 100-1k | 10-100 | 3.5-4.0 |
| 2 | 10-100 | 1-10 | 3.0-3.5 |
| 1 | <10 | <1 | <3.0 |

## 综合评分计算

```
综合分 = 质量×0.30 + 实用性×0.25 + 新鲜度×0.20 + 适配度×0.15 + 社区×0.10
```

展示时四舍五入到0.5分，用星级表示：
- 4.5-5.0 → ⭐⭐⭐⭐⭐ 强烈推荐
- 3.5-4.4 → ⭐⭐⭐⭐ 推荐
- 2.5-3.4 → ⭐⭐⭐ 值得一看
- 1.5-2.4 → ⭐⭐ 参考
- 1.0-1.4 → ⭐ 谨慎
